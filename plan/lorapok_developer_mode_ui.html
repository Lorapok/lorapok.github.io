Full build task list — priority order

Phase 1 Core infrastructure
01
Mode switcher UI — Standard ↔ Developer
Sticky nav toggle that persists mode in localStorage. Developer mode loads a separate route or injects extra sections. Standard mode stays unchanged.
FrontendHigh
02
Firebase project setup — Firestore + Auth + Storage
Initialize Firebase project. Collections: blogs, api_keys (encrypted), comments, subscribers. Enable Google Auth + email for admin panel.
BackendHigh
03
Multi-provider AI key manager
Secure input panel for Claude, OpenAI, Gemini, Mistral, Groq, Cohere. Keys stored encrypted in localStorage (never sent to Firebase). Provider selector dropdown on every AI tool.
FrontendSecurity
Phase 2 AI Blog system
04
Admin panel — blog context editor
Protected admin route. Form fields: topic, key points, target audience, tone. Submit triggers AI generation pipeline. Firebase Auth gate.
BackendFrontend
05
AI blog post generator
Send admin context to chosen AI provider. Stream response to preview pane. Auto-structure into title, intro, sections, conclusion. Save draft to Firestore on completion.
AIBackend
06
AI image generation for blog cover
Auto-generate a cover image from the blog title using Stability AI or fal.ai. Upload to Firebase Storage. Attach URL to blog document. Show in post card thumbnail.
AIStorage
07
Blog listing page + individual post page
Grid of published posts with cover image, title, read-time, tags. Clicking opens full Markdown-rendered post. Slug-based routing. Load from Firestore in real time.
FrontendFirestore
08
Blog admin dashboard
Table of all posts: status (draft / published / scheduled), edit, delete, regenerate. One-click publish. Schedule post for future date using Firestore timestamps.
BackendFrontend
09
Comment system + reactions
Firebase-backed comments per post. Emoji reactions stored per-user. Optional: AI moderation to filter spam before publish.
BackendAI
Phase 3 Developer Mode — AI Labs
10
Multi-provider AI chat (upgrade existing)
Keep Lorapok AI chat but add provider switcher. Each provider uses its own SDK/REST call behind a common wrapper function. Show active model in chat header.
AIFrontend
11
Live API Playground
Code editor (CodeMirror) where devs write prompts + system prompts. Run against any provider. See raw JSON response alongside formatted output. Copy as curl or JS snippet.
AIFrontend
12
Provider comparison tool
Send the same prompt to 2-3 providers simultaneously. Render side-by-side responses with latency, token count, cost estimate. Save comparisons to Firestore.
AIFirestore
13
Prompt library — community shared
Users submit prompts with title, category, provider tag. Stored in Firestore with upvotes. Search + filter by provider/category. One-click to load into Playground.
FirestoreFrontend
Phase 4 Enrichment + polish
14
Email newsletter — auto-send on publish
Subscriber list in Firestore. On blog publish, trigger email via EmailJS or Resend with post summary + link. Unsubscribe link auto-generated.
BackendEmail
15
Analytics dashboard (admin only)
Track page views, blog reads, AI tool usage counts stored in Firestore. Display sparkline charts in admin panel. No external tracking — fully self-hosted.
BackendFrontend
16
Dark / light mode refinement + design system
Extend existing CSS variables to cover Developer Mode sections. Ensure all new components match the Syne + DM Mono aesthetic. Add micro-animations on tab switch.
FrontendDesign
17
SEO + OG image auto-generation
Each blog post gets a dynamic OG image generated from its title + cover. Meta tags auto-populated from Firestore document. Sitemap auto-updated on publish.
FrontendSEO
